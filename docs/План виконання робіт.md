  - [x] Створити компонент для форми (HTML + стилі).
  - [x] Налаштувати валідацію полів (email, пароль).

  - [ ] Показати повідомлення про успішну реєстрацію або помилки.

- [ ] **Реалізувати форму входу у систему**

  - [x] Створити компонент форми входу (HTML + стилі).

- [ ] **Реалізувати форму зміни пароля**

  - [ ] Створити компонент зміни пароля.
  - [ ] Налаштувати перевірку на збіг нового пароля та підтвердження пароля.
  - [ ] Обробити запит до API для зміни пароля.

- [ ] **Реалізувати форму відновлення пароля через email**

  - [ ] Створити компонент введення email для відновлення пароля.
  - [ ] Обробити запит до бекенду для відправки інструкцій на email.

  - [ ] Додати кнопку "Вхід через Google".
  - [ ] Використати Google OAuth API для обробки авторизації.
  - [ ] Отримати і зберегти токен від Google після успішної авторизації.

### Backend

- [ ] **POST /register (реєстрація)**

  - [ ] Створити маршрут для реєстрації нового користувача.
  - [ ] Валідувати email і пароль на сервері.
  - [ ] Хешувати пароль перед збереженням у базу даних (bcrypt).
  - [ ] Повернути JWT токен після успішної реєстрації.

- [ ] **POST /login (вхід)**

  - [ ] Реалізувати маршрут для перевірки email і пароля.
  - [ ] Порівняти введений пароль з хешем (bcrypt).
  - [ ] Повернути JWT токен після успішного входу.

- [ ] **POST /password/change (зміна пароля)**

  - [ ] Реалізувати валідацію старого і нового пароля.
  - [ ] Хешувати новий пароль і оновити запис у базі даних.

- [ ] **POST /password/recover (відновлення пароля)**

  - [ ] Згенерувати токен для відновлення пароля.
  - [ ] Надіслати email з посиланням для відновлення пароля.
  - [ ] Додати маршрут для обробки зміни пароля за токеном.

- [ ] **Аутентифікація через Google**

  - [ ] Інтегрувати Google OAuth на бекенді.
  - [ ] Налаштувати обмін OAuth токеном на JWT для подальшої аутентифікації.

## 2. Головна сторінка (Feed)

### Frontend

- [ ] **Створити компонент для відображення стрічки постів**

  - [ ] Дизайн компонента з постами (картка поста, автор, дата, контент).
  - [ ] Підключити API для отримання постів.
  - [ ] Налаштувати пагінацію або інфініті скролінг.

- [ ] **Реалізувати форму для створення нового посту**

  - [x] Створити компонент форми поста з обмеженням на кількість символів.
  - [ ] Додати можливість форматування тексту (курсив, жирний).
  - [ ] Обробити запит на створення посту через API.

- [ ] **Реалізувати взаємодію з постами**

  - [x] Додати кнопки лайку, дизлайку, коментарів.
  - [ ] Реалізувати оновлення кількості лайків та дизлайків без перезавантаження сторінки (вебсокети або інтервальне оновлення).
  - [x] Додати поле для коментарів і відображення списку коментарів.

### Backend

- [ ] **GET /posts (отримати стрічку постів)**

  - [ ] Реалізувати логіку для отримання постів із бази даних.
  - [ ] Додати підтримку пагінації.

- [ ] **POST /posts (створення посту)**

  - [ ] Додати валідацію полів (текст, кількість символів).
  - [ ] Зберегти новий пост у базу даних і повернути його клієнту.

- [ ] **PUT /posts/:id (редагування посту)**

  - [ ] Реалізувати оновлення існуючого посту.
  - [ ] Перевірити, чи пост належить поточному користувачу перед оновленням.

- [ ] **DELETE /posts/:id (видалення посту)**

  - [ ] Видалити пост із бази даних.
  - [ ] Реалізувати перевірку прав на видалення (чи пост належить користувачу).

- [ ] **POST /posts/:id/like (лайк посту)**

  - [ ] Реалізувати додавання лайку до посту.
  - [ ] Оновити кількість лайків в реальному часі через вебсокети або push-повідомлення.

- [ ] **POST /posts/:id/dislike (дизлайк посту)**

  - [ ] Додати дизлайк до посту та оновити кількість в базі даних.

- [ ] **POST /posts/:id/comment (додати коментар)**

  - [ ] Реалізувати збереження коментаря до посту.
  - [ ] Повернути оновлений список коментарів після додавання.
